分区

  • MBR
    • 第一个扇区
      • 开机记录,分区表(64byte)
      • 最多存储4个分区(3个主分区+1扩展分区)
      • 通过扩展分区分出的更多分区称为逻辑分区
  • GPT
    • 扇区是磁盘的最小存储单位.GPT在定义扇区上使用逻辑区块地址(LBA),默认大小为512bytes
    • GPT没有扩展分区的概念,都是主分区,每个LBA可以分4个分区.
    • 记录分区信息的LBA共33个.第一个记录的是分区表本身的位置,大小,备份分区的位置,CRC32.因此从LBA2~LBA33是直接用于分区的LBA.因此总共可以分128个分区

文件系统

对分区进行格式化是为了在分区上简历文件系统

  • 组成

    • inode: 一个文件占用一个inode,记录文件的属性,记录此文件的内容所在的block编号
    • block: 记录文件的内容,文件太大时,会占用多个block
    • superblock: 整体信息(inode和block的总量,使用量,剩余量)
    • block bitmap: 记录block是否被使用的位图

  • 文件读取

    • Ext2
      • 先在inode查找文件所在的所有block,然后提取
    • FAT文件系统
      • 没有inode,每个block中存储下一个block的编号

  • 磁盘碎片

    • block过于分散,磁头移动距离过大

  • block

    • block分1K,2K,4K三种
    • 一个block只能被一个文件使用,这涉及到根据存储需求选用不同block的问题

  • inode

    • 权限

    • 拥有者与群组

    • 容量

    • 各种时间

    • flag,如SetUID

    • pointer

      • 因为文件往往用很多block,所以有间接,双间接,三简介引用

    • 注意到inode不存储文件名,文件名记录在目录中.所以新增/删除文件这些操作与目录的写权限有关

    • 目录的inode

      • 一个inode
      • 至少一个block
        • 目录下所有文件的inode编号以及文件名

  • 日志

    • 可以用日志修复文件系统

  • 进程状态

    • R: Run
    • D: 不可中断阻塞(IO阻塞)
    • S: 可中断阻塞(等待某个事件完成)
    • Z: Zombie(已经终止但未被父进程获取信息)
    • T: Teminated

  • SIGCHLD

    • 当子一个子进程改变状态时
      • 父进程得到SIGCHLD信号(包括子进程信息(ID,状态,使用CPU时间))
      • waitpid()或者wait()调用会返回
    • 子进程退出时,进程描述符不会立即释放

  • wait()

    • 父进程调用wait()会一直阻塞,直到收到一个SIGCHLD

  • waitpid()

    • 可以通过指定pid表示只关心某个子进程的SIGCHLD信号
    • 可以通过传入WNOHANG使得调用变成非阻塞的

  • 孤儿进程

    • 一个父进程退出,而多个子进程还在运行,它们成为孤儿进程,进而被init进程(pid=1)收养

  • 僵尸进程

    • 如果子进程退出,父进程没有通过wait()或waitpid()获得子进程信息,那么子进程的进程描述符依然保存在系统中,成为僵尸进程
    • 解决办法:杀死父进程

  • 协程: Python中有实现,Java中也有但并不多见(Quasar)

    • 在用户态完成对一个线程的复用,即当两个任务想要同时执行,不需要开两个线程,而是在线程的基础之上通过分时复用的方式运行多个协程,需要改变的是寄存器的数据和协程栈,比起线程切换代价小的多
    • 不过实际上协程只有在等待IO的时候才能重复利用线程.如果某个线程中的协程调用了阻塞IO,将会导致线程切换发生,所以需要异步IO来配合线程

  • 信号量与管程的区别

    • 信号量可以并发,取决于s的初始值
    • P操作在操作之前不知道是否会被阻塞,管程的wait操作一定会阻塞
    • 管程隐蔽了同步的细节,更易于用户维护;而信号量PV操作大量分散到代码的各处,不易维护容易造成死锁

  • Linux线程的内存布局

    • Windows线程的概念与Linux的区别

      假如一个进程包含了N个进程

      for Windows: 存在一个进程描述符,依次轮流指向N个线程

      for Linux: 有N个进程(Linux不明确区分线程和进程),即有N个task_struct数据结构,只是这些数据结构的某些资源项是共享的

    • 资源共享

      • 共享一个进程的资源,包括
        • 内存地址空间
        • 进程基础信息
        • 大部分数据
        • 打开的文件
        • 信号处理
        • 当前工作目录
        • 用户和用户组属性
      • 线程独自拥有的
        • 线程ID
        • 一系列的寄存器
        • 栈的局部变量和返回地址
        • 错误码errno
        • 信号掩码
        • 优先级

    零拷贝

    https://zhuanlan.zhihu.com/p/88789697

    • 定义

      • 应用程序用zero-copy来请求kernel直接把磁盘的数据传输给socket,而不是:

        • 先把磁盘数据复制到内核
        • 再复制到用户空间
        • 再复制到socket缓存中
        • 再复制到网卡设备中
        1
2

        read(file, tmp_buf, len);
write(socket, tmp_buf, len);

    • mmap()

      • 少了一次操作系统内核到应用程序存储空间的缓冲

      • 步骤

        • 磁盘中的数据先通过DMA(direct memory access 直接存储访问,一种无需CPU的参与就可以让外设和系统内存之间进行双向数据传输的硬件机制)被复制到操作系统内核缓冲区中;应用程序和操作系统共享这个缓冲区

        • 应用程序调用write()后,数据从内核缓冲区复制到socket相关的缓冲区

        • 内核的socket缓冲区通过DMA复制到网卡(协议缓存)中

          1
2

          tmp_buf = mmap(file, len);
write(socket, tmp_buf, len);
