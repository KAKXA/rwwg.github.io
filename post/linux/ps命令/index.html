ps 命令总结.

ps -l

ps -l : 仅观察自己bash的相关程序,不会延伸到bash的父进程

范例一:将目前属於您自己这次登陆的 PID 与相关资讯列示出来(只与自己的 bash 有关)
[root@www ~]# ps -l
F S   UID   PID  PPID  C PRI  NI ADDR SZ WCHAN  TTY          TIME CMD
4 S     0 13639 13637  0  75   0 -  1287 wait   pts/1    00:00:00 bash
4 R     0 13700 13639  0  77   0 -  1101 -      pts/1    00:00:00 ps

各列:

  1. F : process flags. 4 表示此程序的权限是 root , 1 表示该子进程只 fork 了而没有 exec .
  2. S : stat. 主要的状态有
  3. R : Running
  4. S : Sleep
  5. D : 不可被唤醒的睡眠状态,可能正在等待IO
  6. T : Stop
  7. Z : Zombie
  8. UID PID PPID
  9. C : CPU 使用率
  10. PRI/NI : Priority/Nice. 代表被 CPU 运行的优先顺序,数值越小代表越快被CPU执行.
  11. ADDR/SZ/WCHAN : ADDR 是 kernel function,指出该程序在内存的哪个部分,如果是个 running 的程序,一般就会显示 - , SZ 代表此程序用掉多少内存,WCHAN 表示目前程序是否运行中,同样的, 若为 - 表示正在运行中.
  12. TTY : 登录者的终端机位置
  13. TIME : 实际花费的CPU时间
  14. CMD : 触发此程序的命令为何

范例二:列出目前所有的正在内存当中的程序:
[root@www ~]# ps aux
USER       PID %CPU %MEM    VSZ   RSS TTY      STAT START   TIME COMMAND
root         1  0.0  0.0   2064   616 ?        Ss   Mar11   0:01 init [5]
root         2  0.0  0.0      0     0 ?        S<   Mar11   0:00 [migration/0]
root         3  0.0  0.0      0     0 ?        SN   Mar11   0:00 [ksoftirqd/0]
.....(中间省略).....
root     13639  0.0  0.2   5148  1508 pts/1    Ss   11:44   0:00 -bash
root     14232  0.0  0.1   4452   876 pts/1    R+   15:52   0:00 ps aux
root     18593  0.0  0.0   2240   476 ?        Ss   Mar14   0:00 /usr/sbin/atd

意义不太显然的栏位分别有:

  1. VSZ : 该 process 使用掉的虚拟内存量
  2. RSS :该 process 占用的固定的内存量
  3. START :该 process 被触发启动的时间
